is deliberately disabling animations and transitions when the page is in the "preload" state.

Why This Exists

The is-preload class is usually added to the <body> before the page fully loads. This is a common technique used to prevent animations from playing immediately (especially when the page loads from a cold start or a refresh). The idea is:

  1. When the page is loading, it disables animations (animation: none !important; and transition: none !important;).
  2. When the page is fully loaded, JavaScript removes is-preload, allowing animations and transitions to behave normally.

When Should It Be Defined?

If you want animations and transitions to always work, then removing that rule is a valid option. However, if you need to control when animations play, a better approach is:

css
body.is-preload * {
	opacity: 0; /* Start elements hidden */
}

body:not(.is-preload) * {
	transition: all 0.5s ease-in-out; /* Enable smooth transitions */
}

This way:

  • Elements start hidden while the page loads.
  • When is-preload is removed (via JavaScript), animations fade in smoothly.

Solution for Your Case

If you want transitions to work properly, change your CSS like this:

css
body.is-preload * {
	opacity: 0;
}

body:not(.is-preload) * {
	transition: all 0.5s ease-in-out;
}
